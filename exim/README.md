# Exim Configuration Files

This directory contains Exim4 mail server configuration files for different deployment targets.

## Structure

- `desktop/00_khms` - Configuration for desktop system (master DNS: desktop.dnstunnel.khms)
- `dns/00_khms` - Configuration for dns system (slave DNS: dns.dnstunnel.khms)
- `mail/00_khms` - Configuration for mail server (mail.cloud)

## Purpose

These configuration fragments define DKIM (DomainKeys Identified Mail) signing parameters for Exim4. They are deployed to `/etc/exim4/conf.d/main/00_khms` on each target machine.

### Key Differences

All configurations use:

- The same private key: `/etc/exim4/dkim/khms.eu.key`
- The same selector: `dns_1`
- The same signing headers: `from:to:cc:date:subject`
- DKIM canonicalization: `relaxed`

But differ in the signing domain:

| System  | Machine                | DKIM_DOMAIN |
| ------- | ---------------------- | ----------- |
| Desktop | desktop.dnstunnel.khms | khms.eu     |
| DNS     | dns.dnstunnel.khms     | khms1.de    |
| Mail    | mail.cloud             | khms1.de    |

## Deployment

These files are deployed automatically by `deploy-zones.sh` when called with the appropriate machine addresses. The deployment script (via SSH) copies these files to `/etc/exim4/conf.d/main/00_khms` on each machine and restarts Exim4.

## DNS Records

For DKIM signatures to validate, the public keys must be published at the appropriate DNS records:

- `dns_1._domainkey.khms.eu` - Public key for khms.eu domain
- `dns_1._domainkey.khms1.de` - Public key for khms1.de domain

Both are generated by `generator.pl` from the `[zone_records]` section of `network.ini`.
